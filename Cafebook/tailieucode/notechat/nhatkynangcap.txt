* **Chuẩn hóa cấu trúc Model:** Phá bỏ file `Entities.cs` khổng lồ (hơn 700 dòng), tách từng thực thể (Entity) ra các file `.cs` riêng biệt (như `NhanVien.cs`, `HoaDon.cs`, `KhachHang.cs`...) và đặt vào thư mục `CafebookModel/Model/ModelEntities` theo đúng tiêu chuẩn thiết kế phần mềm.
* **Gộp bảng dữ liệu:** Xóa bỏ hoàn toàn bảng `NguoiGiaoHang`, gộp chung logic của Shipper vào bảng `NhanVien` để dễ quản lý nhân sự tập trung.
* **Xử lý xung đột Khóa ngoại (Foreign Key):**
    * Sử dụng thuộc tính `[InverseProperty]` trong bảng `NhanVien` để EF Core phân biệt rõ 2 danh sách: Hóa đơn do nhân viên tạo (`HoaDonsTao`) và Hóa đơn do nhân viên giao (`HoaDonsGiao`).
    * Sử dụng `[NotMapped]` để bỏ qua các ánh xạ vòng lặp phức tạp chưa cần thiết (như `PhieuThuongPhatNguoiTaos`, `DeXuatSanPhams`) giúp tăng tốc độ khởi động EF Core.
* **Cấu hình DbContext (`CafebookDbContext.cs`):** Bổ sung đầy đủ các khóa chính phức hợp (`HasKey`) cho các bảng trung gian nhiều-nhiều và cấu hình `DeleteBehavior.NoAction` để tránh lỗi "Cascade Delete" (xóa dây chuyền) trong SQL Server.

### 2. Cải tiến Kiến trúc Server / Backend API (`CafebookApi`)
* **Làm sạch file cấu hình:** Loại bỏ các thông số tĩnh trong `appsettings.json` (như cấu hình gửi Email SMTP, cấu hình khóa VNPay) và chuyển toàn bộ vào lưu trữ tại bảng `CaiDat` trong CSDL. * **Quy hoạch lại `Program.cs`:** Thiết lập lại Middleware, tách biệt rõ ràng chính sách CORS (`AllowAll` cho App/Mobile và `SignalRPolicy` cho Web) để khắc phục triệt để lỗi bảo mật truy cập chéo.
* **Chuẩn hóa DTO (Data Transfer Object):** Tạo thư mục `Shared` và thiết kế bộ DTO chuẩn (`LoginRequest`, `LoginResponse`) dùng chung cho cả Web và App.

### 3. Nâng cấp Hệ thống Đăng nhập & Phân quyền (Authentication & Authorization)
* **Tập trung hóa Controller:** Xây dựng `AuthController.cs` tại thư mục `Shared` để xử lý đăng nhập chung cho toàn bộ nhân sự (Quản lý, Thu ngân, Bếp, Shipper, Quản trị viên) thay vì viết rời rạc.
* **Bảo mật JWT:** Ứng dụng chuẩn bảo mật JSON Web Token (JWT). API không chỉ trả về Token mà còn đính kèm Tên vai trò và một mảng Danh sách Mã Quyền (`Quyen`) để Client tự động ẩn/hiện chức năng.
* **Thiết lập dữ liệu quyền chuẩn:** Cấu hình file SQL khởi tạo dữ liệu với 2 Vai trò chính (Quản lý, Nhân viên) đi kèm 2 nhóm Quyền lớn (`FULL_QL`, `FULL_NV`) để đơn giản hóa quá trình test ban đầu.

### 4. Tối ưu hóa App WPF Desktop (`AppCafebookApi` - Máy POS)
* **Cơ chế Cấu hình Server động:** Loại bỏ việc "hardcode" đường dẫn API (localhost). Xây dựng class `AppConfigManager.cs` để tự động tạo và đọc file `AppConfig.json` nằm cục bộ tại thư mục cài đặt của phần mềm.

### BÁO CÁO TIẾN ĐỘ CẬP NHẬT HỆ THỐNG (GIAI ĐOẠN MỚI)

**1. Tái cấu trúc Kiến trúc Phân quyền (Authorization Architecture)**
* **Chuyển đổi cơ chế cốt lõi:** Nâng cấp từ phân quyền theo Vai trò (RBAC - `VaiTro_Quyen`) sang phân quyền theo cấp độ Cá nhân (UBAC - `NhanVien_Quyen`). Việc này cho phép cấu hình quyền hạn linh hoạt và bảo mật ranh giới tốt hơn cho từng nhân viên cụ thể.
* **Cập nhật Database & Entity Framework:** * Xóa bỏ bảng và các ràng buộc khóa ngoại cũ. Thiết lập bảng `NhanVien_Quyen` mới với kiểu dữ liệu chuẩn hóa (`NVARCHAR`).
  * Cập nhật `CafebookDbContext` và các lớp Entity Models (`NhanVien.cs`, `Quyen.cs`) để ánh xạ lại mối quan hệ 1-N.
* **Tối ưu hóa Luồng Đăng nhập (`AuthController`):** Cập nhật API Login để hệ thống tự động quét và cấp phát Token chứa danh sách quyền hạn riêng biệt của chính nhân viên đang đăng nhập, thay vì lấy quyền chung của vai trò.

**2. Xây dựng mới Module Quản lý Phân quyền (`QuanLyPhanQuyenView`)**
* **Tuân thủ Clean Architecture:** Tách biệt hoàn toàn độc lập 3 lớp gồm `QuanLyPhanQuyenDto.cs` (Data Transfer), `QuanLyPhanQuyenController.cs` (API Logic), và file giao diện WPF (`.xaml` / `.cs`).
* **Thiết kế UI/UX Trực quan:** Xây dựng giao diện chia 2 cột thông minh:
  * **Cột trái (Danh sách):** Hiển thị danh sách nhân sự. Tích hợp bộ lọc theo "Vai trò" và thanh tìm kiếm Real-time (nhập đến đâu lọc đến đó mà không cần gọi lại Server).
  * **Cột phải (Quyền hạn):** Hiển thị danh sách các quyền được render động bằng `ItemsControl`, nhóm theo từng hạng mục (NhomQuyen) với các Checkbox trực quan.
* **Bảo mật giao diện:** Màn hình và các nút truy cập được bảo vệ nghiêm ngặt bằng quyền `QL_PHAN_QUYEN`.

**3. Nâng cấp và Chuẩn hóa Module Quản lý Đơn xin nghỉ (`QuanLyDonXinNghiView`)**
* **Đập đi xây lại theo chuẩn mới:** Tách riêng file `QuanLyDonXinNghiDto.cs` và `QuanLyDonXinNghiController.cs` vào đúng thư mục `QuanLy`, không dùng chung code với luồng của nhân viên.
* **Cải tiến Nghiệp vụ thực tế:** Gỡ bỏ chức năng "Tạo đơn mới" ở phía Quản lý. Quản lý giờ đây đóng vai trò người kiểm duyệt (Chỉ xem chi tiết, nhập Ghi chú phê duyệt và bấm Duyệt/Từ chối/Hủy), đảm bảo tính minh bạch của quy trình nhân sự.
* **Tối ưu hóa Giao diện:** * Khóa các trường nhập liệu (Read-only) khi hiển thị chi tiết đơn.
  * Tích hợp bộ lọc Client-side cho thanh tìm kiếm và ComboBox trạng thái (Chờ duyệt, Đã duyệt, Đã từ chối) giúp lưới dữ liệu phản hồi tức thời.
* **Khắc phục lỗi đứt gãy:** Sửa triệt để các lỗi mất liên kết sự kiện giữa XAML và C# (Lỗi `CS1061` như `Filters_Changed`, `BtnLamMoi_Click`...).

**4. Tối ưu Module Quản lý Nhân viên & Xử lý Bug diện rộng**
* **Tự động hóa cấp quyền:** Tích hợp logic tự động gán bộ quyền mặc định (Cấp `FULL_QL` nếu thêm Quản lý, cấp `FULL_NV` nếu thêm Nhân viên) ngay tại API `Create` nhân viên mới, giúp tài khoản vừa tạo có thể sử dụng được ngay.
* **Xử lý cảnh báo Code (CS8601, CS8602):** Fix triệt để các rủi ro Null Reference Exception trong Controller khi xử lý luồng upload, cập nhật và xóa ảnh đại diện (`AnhDaiDien`) của nhân viên bằng cách sử dụng toán tử null-coalescing.
* **Sửa lỗi UI Control (WPF):** Dọn dẹp lỗi Crash Designer hiển thị (`XDG0012`, `MC4005`) do gán sai thuộc tính `CornerRadius` trực tiếp lên các thẻ `<Button>` và `<TextBox>` trên toàn bộ các file XAML của dự án. Đưa thiết kế bo góc về đúng thẻ `<Border>` bên trong `ControlTemplate`.
